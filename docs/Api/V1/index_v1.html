API URL & Versioning

auth

This endpoint authenticates a user using their UUID, password, and a CAPTCHA token. Platform can require the authentication with Two-Factor, the twoFACode parameter will be asked based on the TWO_FACTOR_AUTH_STATUS setting.

Request Body

{
    "uuid": "user-uuid-example",
    "password": "user-password-example",
    "turnstile_token": "valid_test_token",
    "twoFACode": "02YZR88R"
}

HTTP 200 

{
    "success": true,
    "data": [
        "Authenticated user details and JWT token"
    ]
}

HTTP 400 

{
    "success": false,
    "error": [
        "CAPTCHA validation failed",
        "Missing required fields: uuid, password or turnstile_token",
        "Missing required configuration setting: TWO_FACTOR_AUTH_STATUS",
        "Invalid json format",
        "Invalid user provided. Please verify the user data"
    ]
}

HTTP 401 

{
    "success": false,
    "error": [
        "Two-Factor Authentication is active for this account. Please ensure you provide the correct \n                        authentication code.",
        "Two-Factor Authentication is ENFORCED FOR PORTAL accounts.",
        "Two-Factor Authentication it's required for authentication on the portal. \n                        Please visit DOMAIN to set up 2FA and secure your account.",
        "Invalid credentials"
    ]
}

HTTP 403 

{
    "success": false,
    "error": [
        "User account is not verified!",
        "User account is banned from the system!",
        "Your request cannot be processed at this time due to a pending action. \n                        If your account is active, re-login to complete the action"
    ]
}

HTTP 500 

{
    "success": false,
    "message": [
        "An error occurred: Generic server-side error.",
        "JWT key files are missing. Please ensure both private and public keys exist."
    ]
}

This endpoint registers a new user using their email and password, with CAPTCHA validation via the Turnstile token. It handles user creation, password hashing, and CAPTCHA verification. If the user already exists, it returns a conflict error.

Request Body

{
    "email": "user@example.com",
    "password": "strongpassword",
    "first_name": "John",
    "last_name": "Doe",
    "turnstile_token": "valid_test_token"
}

HTTP 200 

{
    "success": true,
    "data": [
        "Registration successful. Please check your email for further instructions"
    ]
}

HTTP 400 

{
    "success": false,
    "error": [
        "Invalid email format.",
        "Invalid JSON format",
        "Missing required fields: email, password or turnstile_token",
        "CAPTCHA validation failed"
    ]
}
This endpoint triggers a password reset for a local auth account. The user must provide their email and a CAPTCHA validation token. The endpoint verifies if the user has an external auth with "PortalAccount" and "EMAIL" providerId, then proceeds with the password reset if the conditions are met.

Request Body

{
    "email": "user@example.com",
    "turnstile_token": "valid_test_token"
}

HTTP 200 

{
    "success": true,

        "Password reset email sent successfully"

HTTP 400 

{
    "success": false,
    "error": [
        "Invalid email format.",
        "Invalid JSON format",
        "Missing required fields: email or turnstile_token",
        "CAPTCHA validation failed"
    ]
}

This endpoint provides Two-Factor Authentication code only for portal accounts. To be able to request a authentication code the account needs to have setup a 2fa with email or SMS.

Request Body

    "uuid": "user-uuid-example",
    "password": "user-password-example",
    "turnstile_token": "valid_test_token"
}

    "data": [

        "Missing required configuration setting: TWO_FACTOR_AUTH_RESEND_INTERVAL \n                        TWO_FACTOR_AUTH_ATTEMPTS_NUMBER_RESEND_CODE TWO_FACTOR_AUTH_TIME_RESET_ATTEMPTS",
        "Invalid json format",
        "Invalid credentials"
    ]
}

HTTP 401 

{
    "success": false,
    "error": [
        "Invalid credentials",
        "Invalid credentials"
    ]
}

HTTP 403 

{

        "The Two-Factor Authentication (2FA) configuration is incomplete. Please set up 2FA using \n                        either email or SMS",

userAccount

This endpoint deletes the currently authenticated user account. Depending on the authentication method, the request body may require a password (Portal Account), a SAMLResponse (SAML), or an authorization code (Google/Microsoft). The request verifies the provided authentication details before performing the account deletion.
This endpoint requires a valid JWT authentication token.

Request Body

{
    "password": "user-password-example"
}

HTTP 200 

{
    "success": true,
    "data": [
        "User Account was deleted successfully."
    ]
}

HTTP 400 

{
    "success": false,
    "error": [
        "Invalid data: Missing required fields.",
        "Invalid JSON format"
    ]
}

HTTP 401 

{
    "success": false,
    "error": [
        "Invalid Request: JWT Token is invalid!",
        "Invalid credentials: The provided password is incorrect.",
        "Authentication Failed: Unable to validate SAML assertion.",
        "Authentication Failed: Invalid or expired authorization code."
    ]
}

HTTP 403 

{
    "success": false,
    "error": [
        "Unauthorized - You do not have permission to access this resource.",
        "Unauthorized: The SAML assertion email does not match the user account email.",
        "The configured IDP Entity ID does not match the expected value. Access denied.",
        "Invalid Two-Factor Authentication configuration. Please ensure that 2FA is set up using \n                        either email or SMS for this account.",
        "The Two-Factor Authentication (2FA) configuration is incomplete. Please set up 2FA using \n                        either email or SMS.",
        "Invalid account type. Please only use email\/phone number accounts from the portal."
    ]
}

HTTP 404 

{
    "success": false,
    "error": [
        "Invalid Account: User account not found.",
        "Required data from the external service could not be located."
    ]
}

HTTP 500 

{
    "success": false,
    "message": [
        "An error occurred while deleting the user.",
        "An error occurred while communicating with an external service.",
        "An unexpected error occurred. Please try again later."
    ]
}
